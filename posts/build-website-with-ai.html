我如何用 AI 从零搭建这个网站

2026 年 7 月 19 日 · Coco

我不是程序员。写这篇文章的时候，我学习 HTML 和 CSS 的时间并不长， JavaScript 更是刚刚入门。但你现在看到的这个网站——Apple 风格的设计、 响应式布局、右下角的 AI 聊天按钮——都是我完成的。

准确地说，是我和 AI 一起完成的。这篇文章记录真实的过程， 包括走通的部分和踩过的坑。

我的工具组合

  • Cursor：内置 AI 助手的代码编辑器，整个网站的代码都在这里完成
  • HTML / CSS / JavaScript：网站本身只用了这三样，没有框架
  • Git + GitHub：版本管理，每次改动都有记录，改坏了可以回退
  • Vercel：部署平台，代码推送到 GitHub 后自动上线

我是怎么和 AI 协作的

最重要的一条经验：需求写得越具体，AI 给的结果越好。 我每次让 AI 改网站，都会把要求列成清单，比如：保持 Apple 极简风格、 HTML 和 CSS 分离到不同文件、页面要完全响应式、不要破坏现有布局。

模糊的指令（"帮我把网站做好看一点"）得到的结果往往不可控； 具体的指令（"右下角加一个蓝色圆形聊天按钮，鼠标悬停轻微放大"） 几乎一次就能到位。

整个网站就是这样一步步迭代出来的：

  • 先搭出静态的首页结构：导航、Hero 区、关于我
  • 加入技能卡片和项目卡片
  • 做左右分栏的 Hero，放上头像
  • 加 AI 聊天按钮和弹窗（目前是界面占位，真正的对话功能在计划中）
  • 每一步都在手机和电脑上检查显示效果

AI 会犯错，验收是我的责任

AI 不是全对的。让 AI 检查我的代码时，它发现了几个自己之前留下的问题： 平滑滚动的样式写错了位置导致一直没生效、Safari 浏览器需要的兼容前缀没加、 还有一段永远不会用到的死代码。

这给我的启示是：把 AI 当成能力很强但需要验收的合作者， 而不是不会出错的机器。每次改动后自己在浏览器里点一遍、 定期让 AI 复查整个项目，这两个习惯帮我避免了很多问题。

部署比想象中简单

代码托管在 GitHub 上，Vercel 关联仓库之后，每次推送代码都会自动部署。 整个流程走通之后，发布更新只需要三个命令：add、commit、push， 几十秒后改动就出现在线上。
